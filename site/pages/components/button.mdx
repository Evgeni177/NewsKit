# Button

## Overview

A styled button element with customisable size.

## Props

Buttons have a number of props to facilitate a variety of uses. Any prop valid on an Button HTML element as described in [here](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/button#Attributes), is also valid on the Button component.
Custom props are detailed below:

<LegacyBlock marginTop="sizing040">
  <Prop name="children" type="string" required>
    The content of the Button is passed as the child of the component.
  </Prop>
    <Prop
    name="size"
    type="ButtonSize"
    enum={ButtonSize}
    default={ButtonSize.Medium}
  >
    Optional size of the Button: Small, Medium or Large.
  </Prop>
  <Prop name="overrides" type="object">
    If provided, overrides the respective presets for the component and provided elements.
    Here are the overrides that the Button accepts:
    <Prop name="padding" type="PaddingPresetKeys">
      If provided, this overrides the padding of the Button. Otherwise,
      spaceInset020Squish, spaceInset030Squish or spaceInset040Squish is used depending on the `size`
      prop.
    </Prop>
    <Prop name="stylePreset" type="string" default="buttonDefault">
      If provided, this overrides the style preset applied to the Button.
    </Prop>
    <Prop name="typePreset" type="string">
      If provided, this overrides the type preset applied to the Button. Otherwise,
      buttonSmall, buttonMedium or buttonLarge is used depending on the `size` prop.
    </Prop>
    <Prop name="space" type="SizingKeys" default="sizing020">
      If provided, this overrides the space between multiple children in the
      underlying Stack. If less than 2 children are passed, this prop is
      irrelevant.
    </Prop>
    <Prop name="iconSize" type="IconSizeKeys">
      If provided, this overrides the icon size of any child icons. Otherwise,
      iconSize010, iconSize020 or iconSize030 is used depending on the `size` prop. If no icons
      are present in the Button's children then this prop is irrelevant.
    </Prop>
    <Prop name="width" type="SizingKeys | string">
      If provided, this sets a fixed width to the Button. This can be a sizing token
      from the theme, or any CSS length value, e.g. 100% for a full-width element.
    </Prop>
    <Prop name="height" type="SizingKeys | string">
      If provided, this sets a fixed height to the Button. This can be a sizing token
      from the theme, or any CSS length value.
    </Prop>
    <Prop name="minHeight" type="SizingKeys | string">
      If provided, this sets a minimum height to the Button. This can be a sizing
      token from the theme, or any CSS length value. By default, sizing060,
      sizing080 or sizing090 is used depending on the `size` prop.
    </Prop>
    <Prop name="minWidth" type="SizingKeys | string">
      If provided, this sets a minimum width to the Button. This can be a sizing
      token from the theme, or any CSS length value. By default, sizing090,
      sizing100 or sizing110 is used depending on the `size` prop.
    </Prop>
    Refer to the defaults below for the object structure:
    <CodeFromFile path="examples/default-presets/button.tsx" />
  </Prop>
  <Prop name="disabled" type="boolean" default={false}>
    If true, the Button will render the disabled state of the styled preset.
  </Prop>
  <Prop name="isLoading" type="boolean" default="false">
    If true, this will cause the Button to render the loading state of the style
    preset.
  </Prop>
</LegacyBlock>

### Styling

The button & Icon Button components are block level elements with a native HTML button with custom styling.

### Accessibility

Each are fully accessible with the native HTML button used.

#### Colour

Text and background colours must have a 3:1 minimum contrast ratio.
Hover states should have clear contrast change or should include a non colour designator.
