# Tag List

## Overview

The tag list uses multiple [tags](/components/tag) to categorise content.

For example: a story about Meghan Markle may have the following tags associated to it: “Prince Harry”, “Royal Family”, “Duchess of Sussex”, “Fashion”.

### Usage

A tag list can be used anywhere that quick links to related content could be beneficial to a user e.g. article pages or search.

## Props

<LegacyBlock $marginTop="sizing040">

    Optional shape of the Tag components.
  </Prop>
  <Prop
    name="$layout"
    type="TagListLayout"
    enum={TagListLayout}
    default="Center"

    Optional layout of the Tag components. This property dictates how the
    components appear when wrapped onto new lines. It has no effect when no
    wrapping is occurring.
  </Prop>
  <Prop name="$spacing" type="SizingKeys">
    The amount of space added to the right of each tag.
  </Prop>
</LegacyBlock>

## Rationale

### Styling

Tag lists display a group of interactive elements. This differentiates them from standard links and buttons. Tags are designed to be compact and discreet as they are intended to supplement the primary content.

The tag list presents a choice and so should not be used if there is only one option. They should always have a direct relationship to the content they represent.

### Accessibility

#### Keyboard navigation
It is recommended to use an unordered list(ul) and a heading to introduce the purpose of the list.

### SEO

#### Ranking
Tagging generates hub pages which act as valuable catch-all pages, these rank well in search, are automatically updated and generate more backlinks.

#### Navigation
Good for site navigation (for crawlers and users), particularly helpful where site structure is poor (The Times).

#### Descriptive
Tagging helps tell Google what the page content is about, this then enables Google to better surface those pages in search.
