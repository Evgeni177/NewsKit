# getAnimationFromTheme()

## Syntax

<Code>{'getAnimationFromTheme(defaultToken, customProp)(props)'}</Code>

## Overview

Retrieve an animation token from the current theme. Animation tokens are CSS values to control CSS animations.

## Parameters

<LegacyBlock marginTop='sizing040'>
  <Prop name='defaultToken' type='ThemeToken'>
    Is the name of the animation token to be retrieved. If provided, it is used as an object key on the animation section of the theme to get the token value.
  </Prop>
  <Prop name='customProp' type="Exclude<keyof Props, 'theme'>">
    Is the name of the component prop in which an animation token name can be provided by the end-user of the component. If provided, the animation to be retrieved will come from the component prop.
  </Prop>
  <Prop name='props' type='Props'>
    is the props of the component including the theme. If the function is used inside a styled component template literal, or a css block, Emotion.js will call the function first with the component props passed in.
  </Prop>
</LegacyBlock>

## Return value

Returns an animation token if valid attributes are received.

Returns an empty string if invalid attributes or no attributes are provided.

## Example

<CodeFromFile language="text" path="examples/utils/get-animation-from-theme.tsx"/>

In this example the `Paragraph` component will include the animation timing value from `animationDuration040` found in the theme if the `Paragraph` is used with no props:

<Code>{'<Paragraph />'}</Code>

However, it will include styles from `animationDuration050` if the `Paragraph` is used with the `animation` prop and the value passed in is `animationDuration050`:

<Code>{'<Paragraph animation="animationDuration050" />'}</Code>
