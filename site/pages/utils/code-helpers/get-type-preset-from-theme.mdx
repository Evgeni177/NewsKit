# getTypePresetFromTheme()

## Syntax

<Code>{'getTypePresetFromTheme(defaultToken, customProp)(props)'}</Code>

## Overview

Retrieve a type preset from the current theme. A type preset is a collection of typography related styles.

## Parameters

<LegacyBlock marginTop='sizing040'>
  <Prop name='defaultToken' type='ThemeToken'>
    Is the name of the type preset to be retrieved. If provided, it is used as an object key on the type preset theme section to get the token value.
  </Prop>
  <Prop name='customProp' type="Exclude<keyof Props, 'theme'>">
    Is the name of the component prop in which a type preset name can be provided by the end-user of the component. If provided, the type preset to be retrieved will come from the component prop.
  </Prop>
  <Prop name='props' type='Props'>
    Is the props of the component including the theme. If the function is used inside a styled component template literal, or a css block, Emotion.js will call the function first with the component props passed in.
  </Prop>
</LegacyBlock>

## Return value

Returns a type preset if valid attributes are received.

Returns an empty string if invalid attributes or no attributes are provided.

## Example

<CodeFromFile path="examples/utils/get-type-preset-from-theme.tsx"/>

In this example the `Paragraph` component will include the typography styles from `body030` found in the theme if the `Paragraph` is used with no props:

<Code>{'<Paragraph />'}</Code>

However, it will include styles from `subhead050` if the `Paragraph` is used with the `font` prop and the value passed in is `subhead050`:

<Code>{'<Paragraph typePreset="subhead050" />'}</Code>
