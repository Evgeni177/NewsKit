# getShadowFromTheme()

## Syntax

<Code>{'getShadowFromTheme(defaultToken, customProp)(props)'}</Code>

## Overview

Retrieve a shadow token from the current theme. Shadow tokens are CSS values to control an element's [box-shadow](https://developer.mozilla.org/en-US/docs/Web/CSS/box-shadow).

## Parameters

<LegacyBlock marginTop='sizing040'>
  <Prop name='defaultToken' type='ThemeToken'>
    Is the name of the shadow token to be retrieved. If provided, it is used as an object key on the shadow section of the theme to get the token value.
  </Prop>
  <Prop name='customProp' type="Exclude<keyof Props, 'theme'>">
    Is the name of the component prop in which a shadow token name can be provided by the end-user of the component. If provided, the shadow to be retrieved will come from the component prop.
  </Prop>
  <Prop name='props' type='Props'>
    is the props of the component including the theme. If the function is used inside a styled component template literal, or a css block, Emotion.js will call the function first with the component props passed in.
  </Prop>
</LegacyBlock>

## Return value

Returns an shadow token if valid attributes are received.

Returns an empty string if invalid attributes or no attributes are provided.

## Example

<CodeFromFile language="text" path="examples/utils/get-shadow-from-theme.tsx"/>

In this example the `Paragraph` component will include the box shadow value from `shadow020` found in the theme if the `Paragraph` is used with no props:

<Code>{'<Paragraph />'}</Code>

However, it will include styles from `shadow030` if the `Paragraph` is used with the `boxShadow` prop and the value passed in is `shadow030`:

<Code>{'<Paragraph boxShadow="shadow030" />'}</Code>
