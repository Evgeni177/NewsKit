# Get from theme utils

A group of functions used to retrieve token values from theme or component props.
The following functions `getColorFromTheme`, `getSizingFromTheme`, `getBorderFromTheme`, `getShadowFromTheme`, `getMotionFromTheme` share the same annotation, therefore we will generalize them as `the getter function`.
Also, since those functions are used in the same way, we will use only `getColorFromTheme` to build our Syntax and Example sections.

## Overview

Tokens are the foundation of every design system.
The visual representation of every component is formed by combining multiple tokens into presets and group of presets later stored into <Link href="/theming/component-defaults">component defaults object.</Link>
The provided functions, specified above, are used to retrieve the token values from the correspodning getter sections.

## Parameters

<LegacyBlock marginTop="sizing040">
  <Prop name="defaultToken" type="MQ<ThemeToken>">
    The getter function will use this as a default token name to retrieve its value from the corresponding theme section.
    Keep in mind that when 'customProp' is present, 'defaultToken' will be used as a fallback.
  </Prop>
  <Prop name="customProp" type="Exclude<keyof Props, theme>">
    If this property is set, the function will try to resolve the token name from 'props' object using 'customProp' as the property.
    This argument also have higher precedence and if set, will always override `defaultToken`.
  </Prop>
  <Prop name="allowNonThemeValue" type="boolean">
    This prop serves as a flag. When true and a token value can't be resolved from the arguments, instead of returning an empty string - the raw argument value will be returned (`customProp` takes precedence).
    For example, this could allow the prop to take a sizing token, or pixel/percent values.
  </Prop>
  <Prop name="props" type="Props">
    The getter functions are curried and the function returned takes the props of the component (including the theme).
    If the function is used inside a styled component template literal, or a CSS block, Emotion.js will invoke the curried function, passing the component props.
  </Prop>
</LegacyBlock>

## Syntax
<Code>{'(string, string, boolean) -> props -> string'}</Code>
<Code>
  {'getColorFromTheme(defaultToken?, customProp?, allowNonThemeValue?)(props)'}
</Code>

## Return value

Returns a CSS value as a string if valid attributes are received.
Returns an empty string if invalid attributes or no attributes are provided.

This is the most basic example. Here `Paragraph` component will include the color value for token: `blue020` found in the theme.

<Code>{'<Paragraph color="blue050"/>'}</Code>

Although we pass `color` as a prop to `Paragraph`, the end result will be color set to `blue020` value, since our getter function doesnt set `customProp`.

Same code snippet, but we also pass `color` as the second argument to our getter function.
<CodeFromFile path="examples/utils/get-from-theme/two-args.tsx"/>

<Code>{'<Paragraph color="blue050"/>'}</Code>

Now the applied color will result to `blue050` value, because prop tokens have higher precedence.
If for some reason the token value can't be resolved from the theme, the `defaultToken = 'blue020'` will be used as a fallback.

All arguments combined.
<CodeFromFile path="examples/utils/get-from-theme/three-args.tsx"/>

<Code>{'<Paragraph color="blue050"/>'}</Code>

Here our color value is valid token name and this again will apply `blue050` value;

<Code>{'<Paragraph color="foo"/>'}</Code>

Now the value of `defaultToken` will be used as CSS value.
This is happening when `allowNonThemeValue = true` and token value can't be resolved from `customProp` or `defaultToken` (such tokens are not defined in the theme)

Keep in mind that if `allowNonThemeValue` was falsy, the return value would end up being empty string.

## getMediaQueryFromTheme()

Contructs a media query from the provided breakpoint keys. Breakpoints are specified in the theme, there are 4: xs, sm, md, lg.

### Syntax

<Code>{'getMediaQueryFromTheme(minWidth, maxWidth)(props)'}</Code>

### Parameters

<LegacyBlock marginTop="sizing040">
  <Prop name="minWidth" type="BreakpointKeys">
    The min-width for the media query. Can be left falsy if only a max-width is to be used.
  </Prop>
  <Prop name="maxWidth" type="BreakpointKeys">
    The max-width for the media query. Can be left falsy if only a min-width is to be used.
  </Prop>
  <Prop name="props" type="Props">
    Is the props of the component including the theme. If the function is used inside a styled component template literal, or a css block, Emotion.js will call the function first with the component props passed in.
  </Prop>
</LegacyBlock>

### Return value

Returns a media query line, to be used before an opening brace.

Returns an empty screen query (`@media screen`) if no min or max width tokens are provided.

## getDeviceQueryFromTheme()

Target a specific device with media query and apply custom styles to it with either a CSSObject or a SerializedStyles string.

### Syntax

<Code>{'getDeviceQueryFromTheme(Array<Devices>, CSSObject | SerializedStyles)()'}</Code>

### Parameters

<LegacyBlock marginTop="sizing040">

  </Prop>
  <Prop name="cssRules" type="CSSObject | SerializedStyles">

## getStylePresetFromTheme()

Retrieve a style preset values as a css template from the current theme.
For more information on what consists of style preset see doc [here](/theming/style-preset).

### Syntax

<Code>{'getStylePresetFromTheme(defaultToken, customProp, options)(props)'}</Code>

### Parameters

<LegacyBlock marginTop="sizing040">
  <Prop name="defaultToken" type="string">
    Is the name of the style preset token to be retrieved. If provided, it is used as an object key on the style presets section of the theme to get the token value.
  </Prop>
  <Prop name="customProp" type="string">
    Is the name of the component prop in which a style preset token name can be provided by the end-user of the component. If provided, the style preset to be retrieved will come from the component prop.
  </Prop>
  <Prop name="options" type="GetStylePresetFromThemeOptions">
    This prop consists of loading, selected, omitStates and omitStyles options. These are component specific behaviours that can be controlled via either state or passed down from parent as prop.
  </Prop>
  <Prop name="props" type="Props">
    The getter functions are curried and the function returned takes the props of the component (including the theme).
    If the function is used inside a styled component template literal, or a CSS block, Emotion.js will invoke the curried function, passing the component props.
  </Prop>
</LegacyBlock>

### Return value

Returns a template string with style preset css styles.

Returns an empty string if invalid attributes or no attributes are provided.

### Example

<CodeFromFile path="examples/utils/get-from-theme/get-style-preset-from-theme.tsx"/>

In this example the `Paragraph` component will use the sizing value

<Code>{'<Paragraph stylePreset="linkInline" />'}</Code>

In this example the `Paragraph` component will use `linkInline` style preset set in the default theme.

<Code>{'<Paragraph stylePreset="linkInline" loading={true} />'}</Code>

In this example the component has a loading state passed down from the parent component.
If so the `loading` style presets, if present, will override the `base` style presets.

<Code>{'<Paragraph stylePreset="linkInline" isSelected={true} />'}</Code>

In this example the component has a current state passed down from the parent component.
If so the `selected` style presets, if present, will override the `base` style presets.

<Code>{'<Paragraph stylePreset="linkInline" omitStates={["disabled"]} />'}</Code>

In this example the component is using the `linkInline` style preset, without using `disabled` state style presets.

<Code>{'<Paragraph stylePreset="linkInline" omitStyles={["backgroundColor"]} />'}</Code>

In this example the component is using the `linkInline` style preset, while ignorning all `backgroundColor` across all style presets.


## getTypographyPresetFromTheme()

Retrieve a typography preset from the current theme. A typography preset is a collection of typography related styles.

### Syntax

<Code>{'getTypographyPresetFromTheme(defaultToken, customProp, options)(props)'}</Code>

### Parameters

<LegacyBlock marginTop="sizing040">
  <Prop name="defaultToken" type="ThemeToken">
    Is the name of the typography preset to be retrieved. If provided, it is used as an object key on the typography preset theme section to get the token value.
  </Prop>
  <Prop name="customProp" type="Exclude<keyof Props, 'theme'>">
    Is the name of the component prop in which a typography preset name can be provided by the end-user of the component. If provided, the typography preset to be retrieved will come from the component prop.
  </Prop>
  <Prop name="options" type="{withCrop: boolean}">
    When true, will apply the crop options retrieved from the preset.
  </Prop>
  <Prop name="props" type="Props">
    Is the props of the component including the theme. If the function is used inside a styled component template literal, or a css block, Emotion.js will call the function first with the component props passed in.
  </Prop>
</LegacyBlock>

### Return value

Returns a typography preset if valid attributes are received.

Returns an empty string if invalid attributes or no attributes are provided.

### Example

<CodeFromFile path="examples/utils/get-from-theme/get-typography-preset-from-theme.tsx"/>

In this example the `Paragraph` component will include the typography styles from `editorialParagraph030` found in the theme if the `Paragraph` is used with no props:

<Code>{'<Paragraph />'}</Code>

However, it will include styles from `editorialSubheading050` if the `Paragraph` is used with the `font` prop and the value passed in is `editorialSubheading050`:
