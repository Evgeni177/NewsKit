# getMotionFromTheme()

## Syntax

<Code>{'getMotionFromTheme(defaultToken, customProp)(props)'}</Code>

## Overview

Retrieve a motion token from the current theme. Motion tokens are CSS values to control CSS motions.

## Parameters

<LegacyBlock marginTop='sizing040'>
  <Prop name='defaultToken' type='ThemeToken'>
    Is the name of the motion token to be retrieved. If provided, it is used as an object key on the motion section of the theme to get the token value.
  </Prop>
  <Prop name='customProp' type="Exclude<keyof Props, 'theme'>">
    Is the name of the component prop in which a motion token name can be provided by the end-user of the component. If provided, the motion to be retrieved will come from the component prop.
  </Prop>
  <Prop name='props' type='Props'>
    Is the props of the component including the theme. If the function is used inside a styled component template literal, or a css block, Emotion.js will call the function first with the component props passed in.
  </Prop>
</LegacyBlock>

## Return value

Returns a motion token if valid attributes are received.

Returns an empty string if invalid attributes or no attributes are provided.

## Example

<CodeFromFile path="examples/utils/get-motion-from-theme.tsx"/>

In this example the `Paragraph` component will include the motion timing value from `motionDuration040` found in the theme if the `Paragraph` is used with no props:

<Code>{'<Paragraph />'}</Code>

However, it will include styles from `motionDuration050` if the `Paragraph` is used with the `motion` prop and the value passed in is `motionDuration050`:

<Code>{'<Paragraph motion="motionDuration050" />'}</Code>
